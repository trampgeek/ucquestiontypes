Note to would-be editors

This is a very complex question type with a lot of code replicated in the template and prototype extra fields. To edit, download the source code from python3_roboturtle in the ucquestiontypes repository on github. The functionality is broken into 10 or more files, and a build.py program is used to generate the content for the template and prototype extra fields.

About this question type

The RobotTurtle question type uses the Python Turtle to implement a variant of Reeborg's World - see https://reeborg.ca/reeborg.html. Students have to write code that gets the RobotTurtle to perform some task specified in the question text, such as moving to a specified target point, or picking up a number of items and taking them to a specified location.

Commands (all global functions) are:
  • move() - move one cell forward
  • turn_left() - rotate 90 degrees counterclockwise
  • turn_right() - rotate 90 degrees clockwise
  • take() - pick up an item from the current cell and place it in the inventory
  • put() - place the least-recently-picked up object in the current cell
  • toss() - place the least-recently-picked up object in the cell directly in front of the current cell.
  • build_wall() - build a wall directly in front of the robot, i.e. at the edge of the current cell in the direction the robot is facing.
  • print_state() - output the robot's location and heading.
State-checking boolean functions
  • at_goal() - true if the robot is at its final goal position
  • front_is_clear() - true if the robot is able to move forward
  • right_is_clear() - true if the cell to the right is clear
  • wall_in_front() - true if there's a wall in front
  • wall_on_right() - true if there's a wall to the right
  • object_here() - true if there's an object in the current cell
  • carries_object() - true if the inventory is non-empty
  • is_facing_north() - true if the robot is facing north

Authoring RobotTurtle questions

IMPORTANT:
  1. You must check TwigAll or you'll get inexplicable error messages!
  2. You cannot (easily) enter the Answer (or the Preload) directly. You will need to Preview the question, use Ctrl+Alt+m to disable the UI, and copy and paste the answer from the preview back into the authoring form.
Template Parameters
  • testcases - a JSON list of one of more tests as below. REQUIRED.
  • disabledfunctions - a list of the names of the functions from the above list that are not allowed to be used.
The format of a testcase

The question template parameter field must define a parameter testcases, which is a JSON list of one or more tests. Each test is a JSON object with the following structure:

{
  "world_class": "TargetWorld",  # Selected subclass of World - see types of World below
  "world_params": [     # Parameters to pass to World constructor
     "labelled_cells": [],  # Fixed icons (each a [location, item_name] pair). 
  ],
  "grid_size": [10, 12], # [rows, columns]
  "robot_start": # Initial robot position
  {
    "position": [0, 0],
    "heading": "north"
  },
  "walls": [], # Each wall is [[x1, y1], [x2, y2]]
  "blocked_cells": [], # Filled-in cells
  "items": []  # Items.
}

Items is a list of items, each consisting of the form:

{   
  "type": "leaf",  # Item type
  "position": [5, 7],  # Which cell or the value "inventory" 
  "count": 3  # Number of such items at that cell (or inventory)
}

Currently available items types are leaf, key, flag (chequered), house, star, target.

blocked_cells is a list of rectangular regions each specified as a list of 
2 element lists - bottom left hand corner and top right.

An example world:
{
  "grid_size": [12, 8],
  "world_class": "TargetWorld",
  "world_params": {
    "target": [11, 7],
    "cell_size": 70 # Pixels, default is 60.
  },
  "robot_start": {
    "position": [0, 0],
    "heading": "east"
  },
  "walls": [
    [[0, 4], [8, 4]],
    [[8, 4], [8, 0]],
    [[4, 8], [4, 5]]
  ],
  "blocked_cells": [
    [[6, 2], [8, 3]],
    [[0, 0], [1, 1]]
  ],
  "items": [
    {
      "type": "key",
      "position": [3, 2],
      "count": 5
    },
    {
      "type": "leaf",
      "position": [10, 6]
    },
    {
      "type": "star",
      "position": [5, 7],
      "count": 2
    }
  ]
}

Types of World

The subclass of World determines the goal that the user must satisfy to pass the test. 

TargetWorld

A world in which the only goal is for the robot to finish on a specified target location, usually indicated by a chequered flag or a dart-board target.

World Parameters:

target
The 2-element array specifying the robot's target cell (required)
target_icon
The item type used to label the target cell, usually 'flag' (chequered), 'target' (dart-board style), 'home' or 'house' (where the last two are aliases of each other). Optional, default 'flag'

PathWorld

A world in which the robot is required to follow exactly a given path.

World Parameters:

path
Describes the exact sequence of cells the robot must visit from start to end. It is a list of path segments where each segment is a two-element list of 2-element locations denoting the start and end of the segment. Segments are necessarily either horizontal or vertical. Required.

PickUpItemsWorld

A subclass of TargetWorld in which the robot must not only finish at the specified target location but is also required to pick up all items in the grid (specified in the world itself, not the world parameters) except for explicitly listed exceptions. Additionally the robot may be required to park all the collected items at a specified dump location.

World Parameters:

item_dump
The 2-element array specifying where all collected items must be dumped. If not specified it is required only that the items do not appear in any of the grid cells, which implies they are in the robot's inventory.
exclude_cells
A list of 2-element locations of all cells that contain items that the robot does not need to gather up (so are ignored in the check).
